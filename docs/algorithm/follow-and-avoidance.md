# 智能跟随 + 避障算法（算法2 · 闭环版）

本文件描述行李箱「跟随用户 + 自主避障」算法的设计、坐标约定、参数与调参方法。

> **算法2 与算法1 的区别**：算法1 的底盘是「H 桥 + 开环占空比」假设；**算法2 改成你真实的硬件 ——
> APO-DL 电调 + RC 航模 PWM 信号驱动，并用 AB 正交编码器做轮速 PID 闭环、用 IMU 做航向闭环**。
> 上层「跟随 + VFH 避障 + 状态机」沿用算法1 并保持纯 C 可单测，底层执行变成真正的闭环控制：
> 指令 `(v, ω)` 会被实际跟踪，而不再是开环估计。

代码位于：

```
components/control/chassis/          # 闭环底盘：电调 RC-PWM + AB 编码器轮速 PID
components/control/follow_avoid/     # 纯算法（跟随 + 避障 + 状态机），可在 PC 上单测
examples/follow_robot/               # 整机示例：传感器任务 + 控制循环 + IMU 航向闭环 + 底盘
tests/algorithm/                     # 算法 + PID PC 单元测试
```

## 1. 硬件与坐标系

| 部件 | 位置 | 作用 |
|------|------|------|
| 后两轮（电机轮） | 底盘后方 | 差速驱动，提供前进速度 v 与转向角速度 ω；**带 AB 编码器** |
| 前两轮（万向轮） | 底盘前方 | 被动随动脚轮 |
| APO-DL 电调 ×2 | 后两轮 | 收 RC 航模脉冲（50 Hz，1000–2000 µs，1500=停） |
| RPLIDAR C1 | 底盘前方 | 360° 扫描 → 前向 180° 障碍物直方图 |
| 超声波 ×2（A02YYUW） | 箱体前方左右两角 | 近距离安全兜底，补激光雷达盲区/低矮薄障碍 |
| IMU | 底盘中部 | **航向闭环**：用偏航角误差修正转向指令 |
| UWB（BU0x） | 箱体 | 跟随目标：用户随身标签的距离 + 方位 |

**机体坐标系**（贯穿整个算法）：

```
        +x  （前进方向）
         ^
         |
   +y <--+      角度 0   = 正前方 (+x)
  （左）  |      角度 > 0 = 左侧（逆时针 CCW）
         |      角度 < 0 = 右侧（顺时针 CW）

   ω > 0 使机体左转（CCW），与 chassis_set_velocity 一致。
```

后轮差速、前轮万向 → 标准「后轴差速驱动」：瞬时旋转中心在后轮轴上，万向轮自由跟随。

## 2. 软件架构

每个传感器各跑一个 FreeRTOS 任务，把结果写入互斥量保护的快照；固定频率的控制任务读取快照、
运行纯算法、走 IMU 航向闭环、再驱动闭环底盘：

```
 uwb_task   ─┐                        ┌─► chassis_set_velocity(v, ω) ─► chassis_update(dt)
 lidar_task ─┤  互斥量保护的共享快照   │        （电调脉冲 ← 前馈 + 轮速 PID ← AB 编码器）
 ultra_task ─┼──► g_shared ──► control_task ──► fa_update() ──► IMU 航向闭环 ─┘
 (IMU 读取) ─┘     (含时间戳)        (50 Hz)
```

- 任何传感器初始化失败 → 其数据保持「过期/无效」，算法自动降级：
  - 没有激光雷达 → 仅靠超声波做安全停障；
  - 没有 UWB → 进入 SEARCH 旋转找回，超时后 IDLE 停车；
  - **编码器掉线 → 退化为纯前馈开环驱动**（仍能走，只是不再精确）；
  - **IMU 掉线 → 跳过航向闭环**，直接用算法输出的 ω。
- 快照带时间戳，控制任务按新鲜度判断数据是否可用（`*_FRESH_US`）。

`follow_avoid` 与 `chassis` 的运动学/PID 均不依赖任何 ESP/FreeRTOS API（纯 C + math.h），
因此与 PC 单测共用同一份逻辑（见 `tests/algorithm/`）。

## 3. 跟随控制

输入：UWB 解析出的目标 `{距离 d, 方位 β}`。

- **线速度**（保持车距）
  - `d > 期望车距`：`v = kp_dist · (d − 期望车距)`，向前追；
  - 落在 `[期望车距 − 停止带, 期望车距]`：`v = 0`，原地保持，避免抖动；
  - 比停止带还近：`v = 0`。**不倒车** —— 车尾没有任何传感器，盲倒不安全。
- **角速度**（对准目标）：`ω = kp_bear · heading + kd_bear · d(heading)/dt`。
- **耦合**：转角越大，前进越慢（`turn_scale = 1 − |heading| / 90°`），避免边急转边猛冲。

## 4. 避障控制（VFH-lite + 速度调速器 + 超声波兜底）

1. **极坐标直方图**：把激光雷达点按机体角度装进前向 FOV（默认 ±90°，36 个扇区，每扇区 5°），
   每个扇区记录最近障碍距离。
2. **阻塞判定 + 膨胀**：某扇区最近障碍 < `安全距离` 即判为阻塞；再按机体半宽在该距离下张成的
   角宽（`asin(半宽/距离)`）向两侧膨胀，确保不会去钻比车还窄的缝。
3. **航向选择（VFH-lite）**：在所有「空闲」扇区里挑代价最小者：
   `cost = w_goal·|扇区角 − 目标方位| + w_smooth·|扇区角 − 上次航向|`，
   即「尽量朝向用户、又尽量不来回打舵」。选中的扇区角作为转向目标 heading。
4. **速度调速器**：用前向锥内（含两超声波）的最近净空 `clearance` 线性压速 ——
   `clearance` 从 `减速距离` 降到 `急停距离` 时，v 从满速线性压到 0。这是避障的「软」层。
5. **超声波兜底 / 急停**：`clearance ≤ 急停距离`（雷达前锥或任一超声波）→ ESTOP：线速度刹停，
   并朝较空的一侧原地旋转脱困，绝不向障碍物前进。

## 5. 状态机

| 状态 | 触发 | 行为 |
|------|------|------|
| IDLE | 从未见过目标且已超时 | 停车 |
| SEARCH | 目标丢失超过重捕时间 | 朝最后已知方位原地旋转找回；超时退回 IDLE |
| FOLLOW | 目标有效且正前方畅通 | 朝目标跟随 |
| AVOID | 目标有效但正前方被挡 | 绕开（VFH 选侧），减速通过 |
| ESTOP | 前向/超声波进入急停距离，或四面被堵 | 刹停 + 朝空侧旋转脱困 |

急停优先级最高，其次目标丢失处理，最后才是正常跟随/避障。上层输出都经过**加速度限幅**
（升速、降速分别限幅，刹车比加速更快），平滑、保护齿轮。

## 6. 闭环底盘（算法2 核心）

### 6.1 电调 RC-PWM 输出

每个电机由 APO-DL 电调驱动，电调收的是航模舵机脉冲：50 Hz，1000–2000 µs，**1500 µs = 中位/停**，
>1500 前进、<1500 后退。用 LEDC（14 位 @ 50 Hz）生成，与独立的 `动力轮代码` 分支完全一致。

### 6.2 AB 编码器 + 轮速 PID

```
目标轮速 (m/s) ──► 前馈 ff_us = (2000-1500)/Vmax · 目标轮速  ┐
                                                            ├─► 电调脉冲 = 1500 + 限幅(ff + pid)
实测轮速 (m/s) ──► 误差 ──► PID(kp,ki,kd) ──► 修正 us ───────┘
   ▲
AB 编码器四倍频计数 / ticks_per_meter / dt
```

- **前馈**把目标速度直接映射成脉冲，保证「即使编码器掉线也能走」（优雅降级）；
- **PID** 用编码器实测轮速做闭环，修掉前馈误差、负载变化、左右电机不一致；
- PID 输出与积分项**同限幅**（抗积分饱和），微分项取在「测量值」上（避免设定值跳变踢腿）；
- 目标为 0 时强制中位并清积分，电调不蠕动；
- 脉冲再经 `slew_us_per_s` 斜率限幅，保护电流/齿轮。

### 6.3 IMU 航向闭环

`follow_avoid` 输出的 ω 视为「期望转向意图（前馈）」。控制循环把它积分成参考航向 `yaw_ref`，
再用 IMU 实测偏航角 `yaw` 的误差修正：

```
yaw_ref += ω · dt
ω_cmd    = ω + heading_kp · wrap(yaw_ref − yaw)      // 限幅到 ±最大角速度
```

这样即便万向轮拖滞、轮子打滑，机体也能**真正转到**期望角度、直线段能**压住航向**。
仅在 FOLLOW/AVOID（真正跟人）时启用；SEARCH/ESTOP 需要自由旋转，自动关闭并重置参考。

### 6.4 安全失效保护

`chassis_update()` 每周期累计「距上次设定值的时间」，若控制任务卡死、超过
`failsafe_timeout_s`（默认 0.3 s）没有新指令，自动把电调拉回中位停车。上电时先在中位保持
`ESC_ARM_MS`（默认 2 s）让电调解锁。

### 6.5 里程计

底盘顺带用编码器积分出 `(x, y, yaw)` 航位推算，供调试/上层扩展（`chassis_get_odometry`）。

## 7. 参数与调参（menuconfig）

全部参数在 `examples/follow_robot/main/Kconfig.projbuild`，`idf.py menuconfig` 可改。

| 参数 | 默认 | 说明 / 调参方向 |
|------|------|------|
| 期望车距 `FOLLOW_DISTANCE_MM` | 1000 | 跟得更近就调小 |
| 停止带 `STOP_BAND_MM` | 250 | 抖动就调大 |
| 最大线速度 `MAX_LINEAR_MMPS` | 700 | 场地小就调小 |
| 最大角速度 `MAX_ANGULAR_MRADPS` | 1600 | 转向太猛就调小 |
| 急停距离 `EMERGENCY_DIST_MM` | 350 | 撞了就调大 |
| 减速距离 `SLOW_DIST_MM` | 1200 | 想更早减速就调大 |
| 安全距离 `SAFE_DIST_MM` | 600 | 判定阻塞的阈值 |
| 机体半宽 `ROBOT_HALF_WIDTH_MM` | 220 | 改成实际车宽的一半 |
| 轮距 `TRACK_WIDTH_MM` | 300 | **必须**改成实际两后轮间距 |
| 满速轮速 `MAX_WHEEL_SPEED_MMPS` | 800 | 满前进脉冲时实测轮速（设前馈/限幅） |
| **每米脉冲 `TICKS_PER_METER`** | 2000 | **必须标定**：推车 1 m，读日志里编码器增量 |
| 速度环 `SPEED_KP/KI/KD` | 200/300/5 | 抖→减 KP；跟不上目标速→加 KI；超调→加 KD |
| PID 限幅 `SPEED_PID_LIMIT_US` | 400 | PID 单轮最大修正脉冲 |
| ESC 中位/上下限 `ESC_*_US` | 1500/1000/2000 | 按你电调实际行程改 |
| ESC 解锁时长 `ESC_ARM_MS` | 2000 | 电调上电「滴滴」自检所需 |
| 航向闭环 `HEADING_HOLD` | 开 | 关掉则不读 IMU 修航向 |
| 航向增益 `HEADING_KP_MILLI` | 1500（=1.5） | 直线跑偏/转不到位就加，画龙就减 |
| IMU 偏航取反 `IMU_YAW_INVERT` | 关 | 修正方向相反就打开 |
| 左右电机取反 `MOTOR_*_INVERT` | 关 | 某轮转反（且没改线）就打开 |
| 左右编码器取反 `*_ENC_INVERT` | 关 | 前进却计数变小就打开 |

更深的增益（`kp_dist / kp_bear / kd_bear`、加速度限幅、VFH 权重）在 `fa_default_config()` 里，
可在 `build_fa_config()` 覆盖。

## 8. 电机接线（APO-DL 电调 + RC PWM + AB 编码器）

与 `动力轮代码` 分支一致：

```
左电调信号: LEFT_ESC_GPIO  (默认 GPIO4)      右电调信号: RIGHT_ESC_GPIO (默认 GPIO5)
左编码器 A/B: GPIO6 / GPIO7                  右编码器 A/B: GPIO15 / GPIO16
光电编码器供电 3.3–5 V；电调信号线接对应 GPIO，电调电源/电机线按电调说明接。
```

> 某个轮子转反：若已像 `动力轮代码` 那样**手动对调电机蓝白线**，就保持 `MOTOR_*_INVERT=关`；
> 否则打开对应取反开关即可，无需改线。注意取反电机后，对应**编码器方向**可能也要 `*_ENC_INVERT`。

## 10. 上电调试顺序（强烈建议）

1. **垫高轮子空跑**。先确认电调解锁（上电 2 s 内勿给指令）。
2. **标定编码器方向与每米脉冲**：手推 1 m，看日志 `meas v` 符号是否为正、增量是否合理，
   不对就调 `*_ENC_INVERT`，并据增量设 `TICKS_PER_METER`。
3. **标定满速轮速**：给最大前进，量实际轮速，填 `MAX_WHEEL_SPEED_MMPS`。
4. **整定速度 PID**：先只留前馈（KI=KD=0）看跟随误差，再加 KI 消静差、KD 抑超调。
5. **校航向闭环**：直线跑若往一侧偏，调 `HEADING_KP_MILLI`；方向修反则 `IMU_YAW_INVERT`。
6. 确认 FOLLOW/AVOID/ESTOP 切换与转向方向都正确后再落地。

## 11. 安全与限制

- **无后向感知**：算法刻意不倒车，过近只停车。
- **闭环但仍需标定**：`TICKS_PER_METER`、`MAX_WHEEL_SPEED_MMPS`、PID 必须按实车标定，否则
  速度跟踪不准。编码器掉线会自动退化为开环前馈。
- **UWB 坐标约定**：跟随方位由 `Xcm/Ycm` 算得，若左右颠倒，打开 `UWB_LEFT_IS_POS_X`；
  激光雷达零位/旋向用 `LIDAR_FORWARD_DEG / LIDAR_CW` 校准。
- **失效保护**：控制任务卡死会在 0.3 s 内自动停车；但请始终在可断电的情况下首测。
