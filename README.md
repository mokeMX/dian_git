# 动力轮代码 — APO-DL 电调 + 编码器 键盘遥控测试

ESP32-S3 驱动**两路 APO-DL 无刷电调（RC 航模 PWM 信号）**的最小可用程序：用串口键盘 `w/a/s/d` 遥控后轮差速底盘前进/后退/转向，并实时读取 AB 编码器计数。这是「跟随行李箱」**动力系统的硬件验证 / 手动遥控**分支。

> **这个分支是什么**：独立的**动力轮 bring-up**分支，**不含任何传感器或自动算法**，就是把电调、电机、编码器接对、转对、能用键盘开起来。它定下的电调/编码器引脚与驱动方式（RC-PWM + 4× 正交编码），正是 `算法2` 闭环底盘 `components/control/chassis/` 的硬件基础——两者引脚一致。先在这里把电机调通，再上 `算法2` 做闭环跟随。

---

## 功能

- **RC-PWM 电调驱动**：用 LEDC（50Hz，14 位）生成航模舵机脉冲（1000–2000µs，**1500µs=停**）驱动两路 APO-DL 电调
- **键盘遥控**：通过 USB 串口键入 `w/s/a/d/x` 控制前进/后退/左转/右转/停
- **可调速度**：`+`/`-` 调整速度力度（脉冲偏移 `speed_delta_us`）
- **编码器读数**：AB 正交编码器 4× 解码，`c` 查看左右计数（**仅显示，不参与控制**）
- **死人停车**：每条移动指令最多执行 **1.2s** 就自动停（必须持续按键才持续走）

---

## 硬件连接（与 `算法2` 闭环底盘一致）

| 信号 | ESP32-S3 GPIO | 接到 |
|------|---------------|------|
| 左电调信号 S | **GPIO4** | APO-DL **A 通道** 信号 |
| 右电调信号 S | **GPIO5** | APO-DL **B 通道** 信号 |
| 电调信号地 − | GND | APO-DL A/B 通道 −（共地，必接） |
| 左编码器 A / B | GPIO6 / GPIO7 | 电机编码器 |
| 右编码器 A / B | GPIO15 / GPIO16 | 电机编码器 |

- APO-DL A/B 通道的 **+ 不接**（电调信号正端不取电，只给信号 + 地）。
- 编码器供电按模块标称（3.3–5V），共地。
- **电机方向靠硬件接线**：源码里 `LEFT/RIGHT_MOTOR_REVERSE=0`，注释说明「已手动把一边电机蓝白线反接」。若你没反接线、某轮转反，把对应宏改成 1（或对调电机线）。

## 构建与烧录

```bash
idf.py set-target esp32s3
idf.py build
idf.py flash monitor
```

> ⚠️ 控制走的是 **USB-Serial-JTAG**（板子原生 USB 口的控制台），不是 UART0。`idf.py monitor` 连上后**直接按键**即可遥控；用别的串口助手要连到 USB-JTAG 那个口。

## 键盘命令

| 键 | 作用 |
|----|------|
| `w` | 前进（左右轮 = 1500 + speed_delta） |
| `s` | 后退（1500 − speed_delta） |
| `a` | 左转（左轮减、右轮加） |
| `d` | 右转（左轮加、右轮减） |
| `x` | 停（回 1500µs 中位） |
| `+` / `-` | 加/减速（speed_delta 步进 50，范围 100–500） |
| `c` | 打印左右编码器计数 |
| `z` | 打印状态（速度、脉冲、计数、反向标志） |
| `h` | 帮助 |

默认 `speed_delta_us=300` → 前进脉冲 1800µs、后退 1200µs。

## 关键代码导读（`main/main.c`，单文件）

| 区域 | 重点看什么 |
|------|-----------|
| 顶部 `#define` | 引脚（ESC GPIO4/5、编码器 6/7/15/16）、RC 参数（50Hz、1000/1500/2000µs）、`DEFAULT/MIN/MAX_SPEED_DELTA_US`、`COMMAND_TIMEOUT_MS=1200`（死人停车时长）。 |
| `quad_table[16]` + `*_encoder_isr` | **4× 正交编码解码查表**：用「上一状态<<2 | 当前状态」查表增减计数。这张表和 `算法2` 底盘用的是同一套。 |
| `pulse_us_to_duty` / `set_left_pulse` / `set_right_pulse` | 脉冲宽度(µs) → LEDC 占空比的换算，以及通过 `ledc_set_duty`/`ledc_update_duty` 输出。 |
| `motor_forward/backward/turn_left/turn_right/stop` | 差速动作怎么映射成左右脉冲。 |
| `esc_init` | LEDC 定时器/通道配置，**上电立即输出 1500µs 停止**。 |
| `encoder_init` | GPIO 输入 + 上拉 + `ANYEDGE` 中断 + `gpio_isr_handler_add`（A、B 各注册）。 |
| `command_input_init` | `usb_serial_jtag_driver_install`——控制输入来自 USB-JTAG。 |
| `app_main` while 循环 | 读 1 字节按键、分发命令、`motor_running` + `timeout_count` 实现 1.2s 自动停。 |

## ⚠️ 注意事项 / 安全（无刷电机，务必小心）

1. **第一次务必架空轮子**：无刷电机+电调力气大、转速高，落地直接窜出去很危险。先垫高空转确认方向再落地。
2. **电调要先解锁/校准中位**：上电程序立即输出 1500µs。**如果电调被复位过，需要在 1500µs 处重新做一次中位校准**（按电调说明），否则可能不动或乱转。本分支没有显式 arming 延时（`算法2` 才加了 `ESC_ARM_MS`），上电就给 1500，等电调「滴滴」自检完成再操作。
3. **死人停车 1.2s**：按一下只走 1.2s 就停，是**有意的安全设计**，不是 bug——要持续走就持续按键。`x` 立即停。
4. **编码器是开环只读**：本分支**不**用编码器做速度闭环（没有 PID），计数仅供观察/标定。闭环在 `算法2`。
5. **电机转反怎么办**：优先像源码那样**硬件对调电机蓝白线**（保持 `*_MOTOR_REVERSE=0`）；或把对应 `*_MOTOR_REVERSE` 改 1 用软件反向。两者别同时做。
6. **控制口是 USB-JTAG**：连错串口会没反应；用板子的原生 USB 口 + `idf.py monitor`。
7. **共地**：电调信号地、编码器地必须与 ESP32 共地，否则信号不稳。

## 它和其它分支的关系

- `算法2` 的 `components/control/chassis/` 把这里的 **RC-PWM 输出 + 4× 编码器解码**做成了带**轮速 PID 闭环 + 前馈 + 失效保护**的底盘组件，引脚保持一致。本分支可看作那块底盘的「裸机原型 / 手动验证台」。
